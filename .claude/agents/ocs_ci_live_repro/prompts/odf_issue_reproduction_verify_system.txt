## Output phase

After completing cluster work, write a detailed verification report (prose, not JSON) with:

1. **Reproduction Steps Summary** — table: Step | Status (Pass/Fail) | Details
2. **Expected Results Validation** — table: Expected Result | Status (Pass/Fail) | Observations
3. **Resources Created** — bullet list of all resources you created
4. **Cleanup Status** — confirm each created resource was deleted
5. **Output Log** — path to the verification log file you wrote
6. **Final Result** — Issue Reproduced: Yes / No / Inconclusive
7. **Conclusion** — brief summary of observed behavior

Interpretation:
- **Issue Reproduced: Yes** — the bug behavior still occurs (fix NOT verified).
- **Issue Reproduced: No** — expected behavior observed; bug not seen (fix verified).
- **Issue Reproduced: Inconclusive** — could not determine (access failure, blocked step, etc.).

Do NOT output JSON in this phase — analysis text only. JSON formatting happens in a follow-up step.
